# Linux Log Analysis with grep

## Project Overview

In this lab activity, I used Linux command-line tools to search and filter information from log files and user data files. The main goal was to practice using the `grep` command and piping to locate specific information efficiently.

This project demonstrates foundational cybersecurity and SOC analyst skills related to log analysis and Linux system navigation.

---

## Skills Practiced

- Linux command line
- grep command
- File searching
- Log analysis
- Piping commands
- User investigation
- System navigation

---

## Tools Used

- Linux
- Bash Shell
- grep

---

## Tasks Completed

### Task 1: Search for Error Messages

Navigated to the logs directory and searched the `server_logs.txt` file for entries containing the word `error`.

### Task 2: Find Files Containing Specific Strings

Used piping and grep to identify files containing:
- `Q1`
- `access`

### Task 3: Search User Information

Searched user-related files to:
- identify deleted users
- verify if a specific user existed
- investigate department-related entries

---

## Results

- Found 6 error entries in the server logs
- Found 3 files containing `Q1`
- Found 4 files containing `access`
- Confirmed that user `jhill` was deleted from the system
