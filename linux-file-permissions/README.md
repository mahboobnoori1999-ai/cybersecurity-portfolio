# Linux File Permissions Management

## Project Overview

In this lab activity, I examined and managed file and directory permissions in a Linux environment using the Bash shell. The purpose of the lab was to practice authorization management and strengthen foundational cybersecurity skills related to access control.

The activity involved reviewing existing permissions, identifying incorrect or unnecessary access rights, and modifying permissions to improve system security. These tasks simulate real-world administrative and security responsibilities commonly performed by cybersecurity analysts and system administrators.

Managing file permissions is an important part of protecting sensitive information and ensuring that only authorized users have access to specific resources.

---

## Objectives

The main objectives of this lab were:

- Review file and directory permissions
- Identify inappropriate access permissions
- Modify permissions using Linux commands
- Restrict unauthorized access to directories
- Practice Linux authorization management
